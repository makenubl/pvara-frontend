ğŸ“Š HOW IT WORKS

Application Submission Flow:
  1. Candidate fills form â†’ Clicks Submit
  2. Frontend saves application to localStorage
  3. Frontend makes POST request to: http://localhost:5000/api/send-email-template
  4. Backend receives request, loads template, connects to Gmail SMTP
  5. Email is sent to candidate's email address
  6. Candidate receives: "Application Received - [Job Title]"

Status Update Flow:
  1. HR clicks "Shortlist" on application
  2. Frontend updates status in localStorage
  3. Frontend makes POST request with template: APPLICATION_SHORTLISTED
  4. Backend sends email
  5. Candidate receives: "Congratulations! You've been shortlisted"

Error Handling:
  - If backend is down, application still works (frontend-only mode)
  - If email fails to send, system logs error but continues
  - Frontend shows success toast regardless of email status
